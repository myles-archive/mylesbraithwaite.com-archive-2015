Then weekly I take the logs and analyzes them with a script which puts them into a database, which groups the results. If an error happens more than five times in the given week the script creates a ticket in the issue tracker for me to work on later. I log all not found pages by real people (not search engine bots) to try and understand why they navigated to that page. The performance log is how long the system took to generate the page and SQL queries and time it took to execute them.</p> <p><a href="http://blog.vrplumber.com/">Mike</a> took some good <a href="http://pygta.vrplumber.com/exceptional-conditions.html">notes</a> so if you are interested you should definitely check it out.</p>me@mylesbraithwaite.com (Myles Braithwaite)Wed, 19 Aug 2009 00:00:00 -0400http://mylesbraithwaite.com/blog/2009/aug/19/exception-handling-python/PyGTAPythonExceptionProgrammingLogging
