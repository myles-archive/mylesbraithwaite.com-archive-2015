A few hours latter I got the following query in an email:</p> <blockquote> <p>From: Ivan Avery Frey<br> To: Myles Braithwaite<br> Date: Thu, Feb 4, 2010 at 3:24 PM<br> Subject: What's Episteme???</br></br></br></p> <p>Self-explanatory.</p> </blockquote> <p><strong>Episteme</strong> is a wiki engine (more of a wiki library) I am currently working on (I have actually been working on it for the last two years). It was originally a fork of <a href="http://code.google.com/p/yaki/">Yaki</a> (which powers <a href="http://the.taoofmac.com/">The Tao of Mac</a>), then was powered by <a href="http://couchdb.apache.org/">CouchDB</a>, and now is a fork of <a href="http://hatta.sheep.art.pl/">Hatta</a> (not so much as a fork because I am still going to be using Hatta's <code>WikiStorage</code> module).</p> <p>So why am I working on yet-another-wiki-engine?</p> <ul> <li>I like flat files (easy to backup using rsync).</li> <li>I think that <a href="http://en.wikipedia.org/wiki/Relational_database_management_system">Relation databases</a> (like MySQL and PostgreSQL) were not design for wiki documents.</li> <li>I like storing the documents in a Revision Control System so I can edit them in a text editor.</li> <li>I like being able to search.</li> <li>I don't ware the same cloths every day why should I have to use the same markup language.</li> </ul> <p>This is an example of a wiki document:</p> <div class="codehilite"><pre><span class="n">Title</span><span class="o">:</span> <span class="n">Python</span> <span
